# Task 215 — Journey Summary

## red-acceptance (2026-06-25)

**Surprise:** An authenticated `GET /actuator/info` returns 403 whose Problem Detail reads "Access denied: a valid CSRF token is required for this request", even though the request carried the session CSRF cookie+header and GET is normally CSRF-exempt.
**Why:** The path matches no allow-list rule and falls to `.anyRequest().denyAll()` in `SecurityConfig`; the access-denied response is rendered through the CSRF/access-denied message path rather than a generic denyAll message — status (403) and failure type (AssertionError, test expects 200) still matched the RED prediction.
**Impact:** The security step must match `GET /actuator/info` with an authenticated rule ahead of `denyAll`; once matched, the actuator handler runs and the CSRF-exempt GET returns 200 — build-info.properties and git.properties are already generated by the maven step, so actuator config only needs the starter + `management.*` exposure/info settings.
