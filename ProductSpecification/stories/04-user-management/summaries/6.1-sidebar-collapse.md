# Scenario 6.1 — Collapse toggle persists across reload

## align-design (2026-07-05)

**Quirk:** Node 25 ships a native `localStorage` global (Web Storage) that shadows jsdom's `window.localStorage` in vitest and throws (`setItem is not a function`) without a `--localstorage-file` backing path, so the first app code to read `localStorage` (DashboardShell) crashed `home.smoke.test.ts`.
**Where:** `frontend/src/test/local-storage.ts` (`installInMemoryLocalStorage`) wired into `frontend/src/test/setup.ts` with a per-test `clear()`.
**Implication:** any component that touches `localStorage` is testable in jsdom only because of this polyfill — don't remove it; production/Playwright use the real browser storage.

## align-design (2026-07-05)

**Quirk:** `style.css` breached the 200-line hard limit, so its whole `@layer components` block moved to `styles/components.css`, imported right after `@import 'tailwindcss'`; `style.css` now holds only `@theme`.
**Where:** `frontend/src/style.css`, `frontend/src/styles/components.css`.
**Implication:** new semantic `@apply` component classes go in `styles/components.css`, not `style.css`; keep both under 200 lines. `vite build` confirmed `@apply` resolves across the import.
