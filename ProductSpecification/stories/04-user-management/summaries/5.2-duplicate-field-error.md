# Scenario 5.2 — Duplicate login or email shows a field-level error

## red-frontend (2026-07-04)

**Decision:** The FE logic/API layer for admin user creation uses the ubiquitous term "register" (`registerUser`, `RegisterUserRequest`, `RegisterUserError`, `mapRegisterUserErrorToFieldErrors`, `register-user*.ts`), matching the backend (`UserRegistrationService.registerUser`, `RegisterUserCommand`, `UserRegisteredEvent`) and the existing FE component layer (`RegisterUserModal`); the earlier `create-user*` naming was drift and was renamed.
**Why:** DDD ubiquitous language + `frontend-rules` ("API functions: verb+noun matching endpoint (`registerUser`, …)"); the domain concept is registration (triggers the activation-email lifecycle), and "create" came only from the REST transport verb (POST). The URL stays `/api/admin/users` — REST resource URLs are nouns, not part of the ubiquitous-language rename.
**Where applied:** `frontend/src/features/users/logic/register-user*.ts` + `RegisterUserModal.vue`. Acceptance-test infra (`frontend/acceptance/.../create-user-*`, `createUserBackend`, `createUserAsAdmin`) still says "create" — black-box, decoupled from production; left as a smaller separate follow-up.
