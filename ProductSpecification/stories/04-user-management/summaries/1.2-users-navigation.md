# Scenario 1.2 — Users navigation (in-shell) — Journey Summary

## align-design (2026-06-28)

**Decision:** Navigate between admin pages with a nested-route layout — `DashboardShell` hosts a nested `<RouterView>`, `/` keeps the chrome (top bar + sidebar) and `''` renders the extracted `DashboardHome`, `/users` is a real child route; sidebar items are `<RouterLink>`s with route-name active state (chosen over an `activeSection` view-state ref).
**Why:** a real route per page keeps navigation declarative/presentational, gives each admin page a URL, and scales to future Admin Center pages without growing a branching view-state.
**Where applied:** `frontend/src/router/index.ts` (nested children), `DashboardShell.vue`, `DashboardHome.vue`, `features/users/components/UsersPage.vue`; `home.smoke.test.ts` now mounts `App`+router (not `HomePage` directly) so the nested home child renders.
