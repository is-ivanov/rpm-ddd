# Scenario 3.1 (frontend) — Column filter narrows rows client-side — Journey Summary

## red-playwright (2026-06-28)

**Quirk:** Playwright's `test.fail()` absorbs a thrown assertion/error (so the RED test stays "expected-fail") but it does NOT absorb a 30s whole-test timeout — a spec whose only action is `fill()` on a not-yet-existing input hangs to the timeout and fails the build red instead of passing as expected-fail.
**Where:** `frontend/acceptance/tests/frontend/users/users-grid.spec.ts` (and every later `test.fail()`-locked RED spec — Scn 3.2/3.3 reuse this).
**Implication:** always front a `test.fail()` RED spec with a bounded visibility assertion (`assertGridIsVisible()` / `assert*IsVisible()`, 5s `toBeVisible`) so the failure is a fast assertion the marker can absorb, not a 30s timeout.
