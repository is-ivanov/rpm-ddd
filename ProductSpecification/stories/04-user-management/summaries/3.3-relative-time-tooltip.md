# Scenario 3.3 (frontend) — Relative time + absolute-on-hover tooltip — Journey Summary

## red-playwright (2026-06-29)

**Decision:** Freeze time in the E2E with `page.clock.setFixedTime(FIXED_NOW_INSTANT)` as the FIRST action before navigation, not by faking the whole clock.
**Why:** `setFixedTime` fakes `Date.now`/`new Date` only and leaves timers real — safest for a Vue app with HMR/reactivity — so the relative-time label ("17 days ago"…) is deterministic regardless of CI wall-clock; installing it before navigation guarantees the app reads the frozen now from first render.
**Where applied:** `frontend/acceptance/tests/frontend/users/users-grid.spec.ts` (Scn 3.3); the pattern is reusable for any time-dependent E2E.

## red-frontend (2026-06-29)

**Surprise:** `Intl.DateTimeFormat` emits the DST-aware zone abbreviation (CEST/CET) only under the `en-GB` locale; `en-US` returns `GMT+2`, and `Asia/Tokyo` never yields a letter abbreviation (Intl gives `GMT+9`), so the Tokyo rollover edge must assert the date/time shift, not a tz letter.
**Why:** abbreviation availability is locale- and zone-dependent in ICU; empirically probed via Node `Intl` (Node 25 / ICU 78).
**Impact:** `toAbsoluteTooltipParts` must format with `en-GB` to render CEST/CET; any future change to the formatting locale silently drops the abbreviation. Empty/invalid timeZone is not tested — unreachable (the profile enforces a valid IANA id; invalid-zone is the Scn 5.5 web-slice 422).

## green-frontend-api (2026-06-29)

**Quirk:** Making a field `required` in a zod boundary schema obligates the LIVE backend to emit it — `currentUserResponseSchema` gaining a required `timeZone` means `parse` THROWS against a real `/api/auth/me` that omits it (breaking `auth.store.loadMe` → dashboard bootstrap), yet the FE-mocked Playwright E2E stays green because the mock supplies the key.
**Where:** `frontend/src/features/.../current-user.schema.ts`; the backend gap was closed by the deferred "Backend Foundation: GET /api/auth/me returns timeZone" block.
**Implication:** an `endpoints.md`-declared-but-never-scenario'd contract change rots silently; a required FE schema field needs a backing live-backend emit (and an acceptance assertion), or it only breaks in the real app / full-stack journey, never in the mocked suite.
