# Scenario 3.6 — Status column multi-select filter

## align-design (2026-07-07)

**Quirk:** A popover/dropdown positioned `absolute` inside the users grid is clipped by `.table-card`'s `overflow-hidden` (kept for the card's rounded corners), so options hanging below the filter row get cut off.
**Where:** `frontend/src/features/users/components/UsersStatusFilter.vue`, `.table-card` in `frontend/src/styles/components.css`.
**Implication:** any grid popover (dropdown, calendar) must `Teleport to="body"` and position `fixed` from the trigger's `getBoundingClientRect()` — the same pattern TimeCell's tooltip uses; `absolute` will not work.

## demo (2026-07-07)

**Mistake:** Built the status filter panel with `absolute` positioning at align-design; the clipping shipped through design-review and only surfaced when watching the demo.
**Why wrong:** design-review checks placeholder data + control completeness, not overflow clipping of an open popover — a rendered-open-state visual regression it does not catch.
**Correct location/approach:** teleport grid popovers to `<body>` from the start (see the Quirk above).
