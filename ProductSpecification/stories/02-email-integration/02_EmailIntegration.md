# Email Integration

## Brief Description
Replace the no-op email stub with a real SMTP-backed adapter that sends an HTML account-activation email on user registration. Sending is async via the Spring Modulith persisted-event listener, with a scheduler resubmitting failed publications.

## Flow
1. A user registers → `UserRegistrationService` publishes `UserRegisteredEvent`
2. `UserRegisteredEventListener` (`@ApplicationModuleListener`) generates the JWT activation token and calls `EmailNotificationSender.sendActivationToken(email, login, token)`
3. `SmtpEmailNotificationSender` renders the Thymeleaf template, builds a multipart (HTML + plain-text) `MimeMessage`, and sends it via `JavaMailSender`
4. The email contains a clickable link to the frontend activation page: `{app.frontend-base-url}/activate?token={jwt}`
5. On transient SMTP failure the publication stays incomplete in the Modulith registry
6. A `@Scheduled(fixedRate=5s)` job resubmits incomplete publications younger than 24h until delivery succeeds

## Acceptance Criteria
- Registering a user results in exactly one activation email delivered to the registered address
- Email from-address is `RPM Platform <no-reply@rpm-platform.com>` (config-driven) and subject is `Activate your RPM account`
- Email body (HTML + plain-text alternative) contains the activation link `{frontend-base-url}/activate?token={jwt}`
- Sending is async — the registration HTTP response never blocks on SMTP
- A failed send leaves the event incomplete and is resubmitted by the scheduler (every 5s, up to 24h old)
- SMTP credentials, from-address, subject, and frontend base URL are all externalized config

## Validation Rules
| Field | Rule |
|-------|------|
| Recipient email | Provided by `UserRegisteredEvent` (already validated at registration) |
| Activation token | JWT generated by existing `JwtActivationTokenGenerator` (unchanged) |
| From-address | From `app.mail.from` config — never hardcoded |
| Frontend base URL | From `app.frontend-base-url` config — never hardcoded |

## Screen States
None — backend/integration story. The activation link targets Story 1's existing frontend activation page.

## Core Requirements
- Transport: Spring `JavaMailSender` over SMTP (`spring-boot-starter-mail`), provider chosen at deploy time via `spring.mail.*` — provider-agnostic adapter
- Email body: HTML via Thymeleaf (`spring-boot-starter-thymeleaf`) + plain-text alternative (multipart)
- `SmtpEmailNotificationSender` replaces `NoOpEmailNotificationSender` as the primary/sole bean
- Failure resilience: Modulith JDBC persisted events + resubmit scheduler (`@EnableScheduling`), 24h age cutoff via `IncompleteEventPublications`
- Reuse unchanged from Story 1: `UserRegisteredEvent`, `UserRegisteredEventListener`, `EmailNotificationSender` port, token generators, registration service, Modulith registry
- Acceptance tests use reusable Mailpit infrastructure (shared-first, container-fallback) mirroring the existing DB test pattern
